# Biến và hằng 
 Biến trong "biến thiên" có thể thay đổi được 
 Hằng là giá trị không thay đổi được
 Khai báo hằng: <từ khóa> <Tên hằng> = <Giá trị>
# Kiểu dữ liệu: 
Một biến/hằng luôn có kiểu dữ liệu
Kiểu dữ liệu là loại dữ liệu mà biến mang theo 
# Có 2 nhóm kiểu dữ liệu: 
Kiểu nguyên thủy: Number, String, Boolean
Kiểu tham số: Object
# Toán tử so sánh 
Dùng để so sánh 2 toán hạng, kết quả trả về dạng Boolean 
Phân biệt so sánh hai bằng và ba bằng: 
- 2 bằng: So sash giá trị sau khi chuyển đổi
- 3 bằng: so sánh giá trị và kiểu dữ liệu không chuyển đổi
# Toán tử logic
dùng để kết hợp nhiều điều kiện và trả về boolean
- && (AND): trả vè đúng nêu 2 vế của mệnh đề đung
- || (OR): trả về đúng nếu một trong hai vế đúng
# Toán tử 1 ngôi
Là toán tử chỉ dùng một toán hạng để thực hiện
- Prefix: là toán tử ở phía trước, tăng trước trả về sau 
- Pofix: là toán tử ở phía sau, tăng sau trả về trước 
# Toán tử toán học
Là các toán tử cộng, trừ, nhân, chia 
Khi chia cho 0 sẽ trả về kết quả infinity
# Câu điều kiện 
Dùng để kiểm tra một đoạn logic trước khi chạy, nếu điều kiện đúng thì mới chạy 
Có các loại câu điều kiện: 
*If*  
*If...else*  
*If...else if...else*  
*Switch...case*
## Vòng lặp 
Dùng để lặp lại 1 đoạn logic, có thể lặp lại một số lần nhất định, hoặc lặp vô hạn, tùy theo điều kiện dòng 
Có các loại vòng lặp: 
*for (i)*  
*for (of)*  
*forEach*  
*for (in)*  
*while*  
*do...while*
# Markdown
Là một ngôn ngư đánh dấu văn bản 
Cho phép định dạng văn bản bằng cách sử dụng các ký tự dơn giản, dễ học, dễ đọc
**Tại sao nên học Markdown?**
Tạo document cho automation framework 
Có thể viết tài liệu HDSD framework, API document
**Cú pháp:**
1. Tiêu đề (Heading)
# Tiêu đề cấp 1 → H1

## Tiêu đề cấp 2 → H2

### Tiêu đề cấp 3 → H3

Tối đa đến ###### (H6)

2. Đoạn văn (Paragraph)
Tách đoạn bằng một dòng trống.

Không cần thụt lề bằng dấu cách hoặc tab.

3. Định dạng chữ
In đậm: **text** hoặc __text__ → text

In nghiêng: *text* hoặc _text_ → text

Đậm + nghiêng: ***text*** hoặc ___text___ → text

Gạch ngang: ~~text~~ → text

4. Danh sách
Không thứ tự:

Code
- Mục 1
- Mục 2
Có thứ tự:

Code
1. Mục 1
2. Mục 2
Checklist:

Code
- [x] Hoàn thành
- [ ] Chưa xong
```  

5. Liên kết & Hình ảnh
Liên kết: [Tên hiển thị](https://hoctest.com)

Hình ảnh: ![Alt text](image.jpg)

6. Trích dẫn (Blockquote)
Code
> Đây là trích dẫn
>> Trích dẫn lồng nhau
```
